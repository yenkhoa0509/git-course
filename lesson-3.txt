1. Giải thích các câu lệnh:
git commit --amend -m”<message>” để thay đổi nội dung commit gần nhất
git restore --staged <file> để gỡ file khỏi vùng staging
git reset HEAD~1 để đưa commit gần nhất về lại working directory

2. Hãy liệt kê các file theo từng vùng, liệt kê các commit và message tương ứng sau khi thao tác một loạt các lệnh sau
a. Tạo 3 file: file1, file2, file3
Working directory: file1, file2, file3
Staging Area: Empty
Repository: Empty

b. Chạy lệnh: git init
Working directory: file1, file2, file3
Staging Area: Empty
Repository: Empty

c. Chạy lệnh: git commit -m "init project"
Working directory: file1, file2, file3
Staging Area: Empty
Repository: Empty

d. Chạy lệnh: git add file1
Working directory: file1, file2, file3
Staging Area: file1
Repository: Empty

e. Chạy lệnh: git commit -m "add file"
Working directory: file1, file2, file3
Staging Area: Empty
Repository: file1

f. Chạy lệnh: git add .
Working directory: file1, file2, file3
Staging Area: file2, file3
Repository: file1

g. Chạy lệnh: git restore --staged file2
Working directory: file1, file2, file3
Staging Area: file3
Repository: file1

h. Chạy lệnh: git commit -m "add all files"
Working directory: file1, file2, file3
Staging Area: Empty
Repository: file1, file3

i. Chạy lệnh: git reset HEAD~1
Working directory: file1, file2, file3
Staging Area: file3
Repository: file1
